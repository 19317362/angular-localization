angular-localization

A complete AngularJS localization service with core service and accompanying filters, directives etc.

i18n Directive

The localization key can be defined as the value of the i18n attribute. If the attribute value is not a valid token, then it will itself be used as the element's content.

Attribute localizations cannot contain HTML tags

Localizations defined by the i18n attribute cannot contain HTML tags, as the translation result will be assigned as text, not as HTML. This limitation enables a slightly faster localization, as no sanitization is required.

Basic example

Attach the data-i18n attribute to the element whose text should be localized.

<div class="panel panel-default">
    <div class="panel-body" data-i18n="demo.basicExampleText"></div>
</div>

Dynamic user data

The i18n directive observes all non-directive data-* attributes.

<div class="panel panel-default">
    <div class="panel-body" data-i18n="demo.dynamicExampleText" data-name="{{ user.name }}"></div>
</div>

i18nAttr Directive

It works the same way as the original i18n directive, but instead of updating the element content, it updates their associated HTML attribute(s).

Input Placeholders

<form class="form-horizontal" role="form">
    <div class="form-group">
        <label for="exampleInputEmail" class="col-sm-2 control-label" data-i18n="demo.emailAddress"></label>
        <div class="col-sm-10">
            <input type="email" class="form-control" id="exampleInputEmail" data-i18n-attr="{placeholder: 'demo.emailPlaceholder'}">
        </div>
    </div>
    <div class="form-group">
        <label for="exampleInputPassword" class="col-sm-2 control-label" data-i18n="demo.password"></label>
        <div class="col-sm-10">
            <input type="password" class="form-control" id="exampleInputPassword" data-i18n-attr="{placeholder: 'demo.password'}">
            </div>
        </div>
        <div class="form-group">
            <div class="col-sm-offset-2 col-sm-10">
            <div class="checkbox">
                <label>
                    <input type="checkbox">
                    {{ 'demo.rememberMe' | i18n }}
                </label>
            </div>
        </div>
    </div>
    <div class="form-group">
        <div class="col-sm-offset-2 col-sm-10">
            <button type="submit" class="btn btn-default" data-i18n="demo.submit"></button>
        </div>
    </div>
</form>

Form data will remain when changing languages

Your form data will remain intact when switching back and forth between languages. The page does not and never will refresh because all changes are handled in place. So what are you waiting for — go ahead and try it out!

locale Service

The locale service is equivalent to the i18n directive in all aspects and can be used to generate localized results in situations where the directive cannot be used.

Race conditions

Always use locale.ready() passing in the filename(s) in which the token(s) exists in that context, and enclose any other relevant calls to locale.getString() within this promise's callback function block. Failure to do so may cause unexpected (empty) results due to the asynchronous nature of your entire JavaScript application.

You may use locale.getPath() to find the filename for any given token.

Within locale.ready

Wrap in a call to locale.ready() to ensure that the value will be updated if and only if the filepath has been loaded into memory.

locale.ready('common').then(function() {
    $scope.strings.yes = locale.getString('common.yes');
});
  • {{ strings.yes }}
<div class="dropdown">
    <button class="btn dropdown-toggle sr-only" type="button" id="dropdownMenu1" data-toggle="dropdown">
        Dropdown
        <span class="caret"></span>
    </button>
    <ul class="dropdown-menu" role="menu" aria-labelledby="dropdownMenu1">
        <li role="presentation"><a role="menuitem" tabindex="-1" href="#">{{ strings.yes }}</a></li>
    </ul>
</div>

Without locale.ready

Missing locale.ready() causes the value to be updated immediately, before the service has time to fetch the file.

$scope.strings = {
    no: locale.getString('common.no')
};
  • {{ strings.no }}

        <li role="presentation"><a role="menuitem" tabindex="-1" href="#">{{ strings.no }}</a></li>
    </ul>
</div>

i18n Filter

The i18n filter provides the same functionality as the service. It can be useful in templates where the localization strings are dynamic, e.g. for error messages

Simple Example

Pipe a generated error message to be processed.

{{ messages.generic | i18n }} 
<div class="alert alert-danger">
    <span class="glyphicon glyphicon-remove"></span>
    {{ messages.error | i18n }}
</div>

Parameterized Arguments

Give it an array, map (object) or even a single string

{{ messages.targeted | i18n:'test.json' }}
{{ messages.tests | i18n:filtArgs.tests }}
{{ messages.copyright | i18n:filtArgs.copyright }}
{{ messages.version | i18n:filtArgs.version }} 
<div class="alert alert-danger">
    <span class="glyphicon glyphicon-remove"></span>
    {{ messages.targeted | i18n:'test.json' }}
</div>
<div class="alert alert-success">
    <span class="glyphicon glyphicon-ok"></span>
    {{ messages.tests | i18n:filtArgs.tests }}
</div>
<div class="alert alert-info">
    {{ messages.copyright | i18n:filtArgs.copyright }}
</div>
<div class="alert alert-info">
    <span class="glyphicon glyphicon-info-sign"></span>
    {{ messages.version | i18n:filtArgs.version }}
</div>

Special Use-case

Use a delimiter when it is not possible to follow the standardized conventional syntax for passing in additional information. Because of the way that is has been designed, it will not work in markup and must be called programatically i.e. either locale.getString() or $filter('i18n') and the string following the separator has to be valid JSON formatted.

{{ specials.targeted }}
{{ specials.tests }}
{{ specials.copyright }}
{{ specials.version }} 
<div class="alert alert-danger">
    <span class="glyphicon glyphicon-remove"></span>
    {{ specials.targeted }}
</div>
<div class="alert alert-success">
    <span class="glyphicon glyphicon-ok"></span>
    {{ specials.tests }}
</div>
<div class="alert alert-info">
    {{ specials.copyright }}
</div>
<div class="alert alert-info">
    <span class="glyphicon glyphicon-info-sign"></span>
    {{ specials.version }}
</div>

Using ngPluralize

Partially compatible with native ngPluralize directive.

Limitations

Full compatibility cannot be reached due to limitations of the ng-framework. Right now it is impossible to place the data-count attribute anywhere in the resultant string. There exists a known workaround using $filter('i18n') or locale.getString() directly in your controller or else you could expose a getPlurals() function to the ngPluralize directive. The newly created function should internally address the value of count.

This is somewhat wieldy and cumbersome as it requires you to track offsets manually, the same can also be achieved thru the i18n directive albeit with one extra degree of complexity; the caveat being that of interpolation along with $watchers and/or ng-bind

In such situations it is good to note that locale.ready() is not mandatory as angular's page $digest cycle will fire periodically and take care of it for you.

Also, strings may not be passed to the i18n filter(s) of the ngPluralize directive as is, in the markup.

Events Emitted

The base service exposes certain events which may be listened to, captured and acted upon. There are part of the ngLocalize.Events module which must first be injected as a dependency either at the application-level or in your controller/services enclosing module. Since these are emitted by $rootScope they are available to all descendent $scopes in controllers, services and/or directives.

On Resource Updates

You must listen for localeEvents.resourceUpdates.

angular.module('docs.demo.myModule', ['ngLocalize.Events'])
    .controller('demoControl', ['localeVersion',
        function() {
            $scope.$on(localeEvents.resourceUpdates, function () {
                console.info('ngLocalize: Resource Updated.')
            });
        }
    ]);

On Locale Changes

Triggered by localeEvents.localeChanges.

angular.module('docs.demo.yetAnotherModule', ['ngLocalize.Events'])
    .controller('docsControl', ['localeVersion',
        function() {
            $scope.$on(localeEvents.localeChanges, function (newLocale) {
                console.info('ngLocalize: Locale Changed to ' + newLocale)
            });
        }
    ]);

Custom Configuration

All overridable configuration options are part of localeConf within the ngLocalize.Config module. This has been done to achieve maximum compatibility and configurability with all new as well as any existing small-medium-large-scale applications.

basePath languages

The folder off the root of your wb app where the resource files are located (it can also be used as a relative path).

defaultLocale en-US

The locale that your app is initialized with for a new user.

sharedDictionary common

Commonly occuring words, phrases, strings etc. are stored in this file (it is used to check whether the service itself is ready or not as it is loaded during bootstrap).

fileExtension .lang.json

The extension for all resource files spanning across all languages.

persistSelection true

Whether to save the selected language to cookies for retrieval on later uses of the app.

cookieName COOKIE_LOCALE_LANG

Works in conjuntion with persistSelection and provides the cookie name for storage.

observableAttrs \^data-(?!ng-|i18n)\

A regular expression which is used to match which custom data attibutes may be watched by the i18n directive for 2-way bindings to replace in a tokenized string.

delimiter ::

The delimiter to be used when passing params along with the string token to the service, filter etc.

angular.module('docs.demo', ['ngLocalize.Config'])
    .value('localeConf', {
        basePath: 'languages',
        defaultLocale: 'languages',
        sharedDictionary: 'common',
        fileExtension: '.lang.json',
        persistSelection: true,
        cookieName: 'COOKIE_LOCALE_LANG',
        observableAttrs: new RegExp('^data-(?!ng-|i18n)'),
        delimiter: '::',
    });

Overriding defaults

If you would like to override at least one particular option, then you must explicitly set any other options explicitly to their default values.
