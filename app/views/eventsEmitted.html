Events Emitted

The base service exposes certain events which may be listened to, captured and acted upon. There are part of the ngLocalize.Events module which must first be injected as a dependency either at the application-level or in your controller/services enclosing module. Since these are emitted by $rootScope they are available to all descendent $scopes in controllers, services and/or directives.

On Resource Updates

You must listen for localeEvents.resourceUpdates.

angular.module('docs.demo.myModule', ['ngLocalize.Events'])
    .controller('demoControl', ['localeVersion',
        function() {
            $scope.$on(localeEvents.resourceUpdates, function () {
                console.info('ngLocalize: A resource was recently updated.')
            });
        }
    ]);

On Locale Changes

Triggered by localeEvents.localeChanges.

angular.module('docs.demo.yetAnotherModule', ['ngLocalize.Events'])
    .controller('docsControl', ['localeVersion',
        function() {
            $scope.$on(localeEvents.localeChanges, function (newLocale) {
                console.info('ngLocalize: Locale Changed to '  + newLocale)
            });
        }
    ]);
