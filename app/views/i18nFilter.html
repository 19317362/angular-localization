i18n Filter

The i18n filter provides the same functionality as the service. It can be useful in templates where the localization strings are dynamic, e.g. for error messages

Simple Example

Pipe a generated error message to be processed.

{{ messages.generic | i18n }} 
<div class="alert alert-danger">
    <span class="glyphicon glyphicon-remove"></span>
    {{ messages.error | i18n }}
</div>

Parameterized Arguments

Give it an array, map (object) or even a single string

{{ messages.targeted | i18n:'test.json' }}
{{ messages.tests | i18n:filtArgs.tests }}
{{ messages.copyright | i18n:filtArgs.copyright }}
{{ messages.version | i18n:filtArgs.version }} 
<div class="alert alert-danger">
    <span class="glyphicon glyphicon-remove"></span>
    {{ messages.targeted | i18n:'test.json' }}
</div>
<div class="alert alert-success">
    <span class="glyphicon glyphicon-ok"></span>
    {{ messages.tests | i18n:filtArgs.tests }}
</div>
<div class="alert alert-info">
    {{ messages.copyright | i18n:filtArgs.copyright }}
</div>
<div class="alert alert-info">
    <span class="glyphicon glyphicon-info-sign"></span>
    {{ messages.version | i18n:filtArgs.version }}
</div>

Special Use-case

Use a delimiter when it is not possible to follow the standardized conventional syntax for passing in additional information. Because of the way that is has been designed, it will not work in markup and must be called programatically i.e. either locale.getString() or $filter('i18n') and the string following the separator has to be valid JSON formatted.

{{ specials.targeted }}
{{ specials.tests }}
{{ specials.copyright }}
{{ specials.version }} 
<div class="alert alert-danger">
    <span class="glyphicon glyphicon-remove"></span>
    {{ specials.targeted }}
</div>
<div class="alert alert-success">
    <span class="glyphicon glyphicon-ok"></span>
    {{ specials.tests }}
</div>
<div class="alert alert-info">
    {{ specials.copyright }}
</div>
<div class="alert alert-info">
    <span class="glyphicon glyphicon-info-sign"></span>
    {{ specials.version }}
</div>
